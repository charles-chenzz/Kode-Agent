## 二、消息类型

### 2.1 消息类型定义

```typescript
// src/app/query.ts

// 用户消息
export type UserMessage = {
  type: 'user'
  uuid: UUID
  message: MessageParam  // { role: 'user', content: string | ContentBlock[] }
  toolUseResult?: FullToolUseResult  // 如果是工具结果
  options?: {
    isKodingRequest?: boolean
    kodingContext?: string
    isCustomCommand?: boolean
    commandName?: string
    commandArgs?: string
  }
}

// AI 响应消息
export type AssistantMessage = {
  type: 'assistant'
  uuid: UUID
  costUSD: number
  durationMs: number
  message: APIAssistantMessage
  isApiErrorMessage?: boolean
  responseId?: string  // 用于 Responses API 的响应链
}

// 进度消息（用于实时更新 UI）
export type ProgressMessage = {
  type: 'progress'
  uuid: UUID
  toolUseID: string
  content: AssistantMessage
  normalizedMessages: NormalizedMessage[]
  siblingToolUseIDs: Set<string>
  tools: Tool[]
}

// 联合类型
export type Message = UserMessage | AssistantMessage | ProgressMessage
```

## 四、消息标准化

### 4.1 消息标准化流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       消息标准化流程                                        │
│                                                                             │
│   normalizeMessagesForAPI(messages)                                         │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  1. 过滤消息                                                        │  │
│   │                                                                     │  │
│   │     • 跳过 ProgressMessage (不发送给 API)                           │  │
│   │     • 跳过 isApiErrorMessage (API 错误消息)                         │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  2. 合并相邻消息                                                    │  │
│   │                                                                     │  │
│   │     相邻的 UserMessage 合并为一个：                                 │  │
│   │     • tool_result 块放在前面                                       │  │
│   │     • 其他块放在后面                                               │  │
│   │                                                                     │  │
│   │     相邻的 AssistantMessage (同一 message.id) 合并：                │  │
│   │     • 合并 content 数组                                            │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  3. 标准化内容格式                                                  │  │
│   │                                                                     │  │
│   │     string content → [{ type: 'text', text: content }]             │  │
│   │     array content  → 保持原样                                      │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  4. 返回标准化后的消息列表                                          │  │
│   │                                                                     │  │
│   │     (UserMessage | AssistantMessage)[]                              │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 消息排序规则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         消息排序规则                                        │
│                                                                             │
│   reorderMessages(normalizedMessages)                                       │
│                                                                             │
│   规则：确保消息按照逻辑顺序排列                                            │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   1. 工具调用消息在前                                               │  │
│   │      AssistantMessage { type: 'tool_use', ... }                     │  │
│   │                                                                     │  │
│   │   2. 进度消息紧随其后                                               │  │
│   │      ProgressMessage { toolUseID: matching_id, ... }                │  │
│   │                                                                     │  │
│   │   3. 工具结果消息最后                                               │  │
│   │      UserMessage { type: 'tool_result', tool_use_id: matching_id }  │  │
│   │                                                                     │  │
│   │   示例：                                                            │  │
│   │   ┌─────────────────────────────────────────────────────────────┐  │  │
│   │   │ [                                                            │  │  │
│   │   │   AssistantMessage: "让我读取文件",                           │  │  │
│   │   │   AssistantMessage: tool_use Read,                           │  │  │
│   │   │   ProgressMessage: "正在读取...",                             │  │  │

## 六、自动压缩 (Auto-Compact)

### 6.1 压缩触发条件

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       自动压缩机制                                          │
│                                                                             │
│   问题：消息列表会越来越长，超过模型上下文限制                              │
│   解决：自动压缩旧消息，保留关键信息                                        │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  触发条件检查                                                       │  │
│   │                                                                     │  │
│   │  checkAutoCompact(messages, toolUseContext)                         │  │
│   │                                                                     │  │
│   │   条件：                                                            │  │
│   │   • 消息数量超过阈值                                                │  │
│   │   • Token 数量接近上下文限制                                        │  │
│   │   • 用户手动触发                                                    │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                    │                                        │
│                                    ▼ 需要压缩                               │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  压缩策略                                                           │  │
│   │                                                                     │  │
│   │   1. 保留最近 N 条消息（不压缩）                                    │  │
│   │   2. 对旧消息生成摘要                                               │  │
│   │   3. 替换旧消息为摘要消息                                           │  │
│   │                                                                     │  │
│   │   压缩前：                                                          │  │
│   │   [msg1, msg2, msg3, ..., msg100]                                   │  │
│   │                                                                     │  │
│   │   压缩后：                                                          │  │
│   │   [摘要消息, msg95, msg96, ..., msg100]                             │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                    │                                        │
│                                    ▼                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  摘要生成                                                           │  │
│   │                                                                     │  │
│   │   调用 AI 模型生成摘要：                                            │  │
│   │                                                                     │  │
│   │   prompt: `                                                         │  │
│   │     请总结以下对话的关键信息：                                      │  │
│   │     - 讨论了什么话题                                                │  │
│   │     - 做了什么操作                                                  │  │
│   │     - 重要决策和结论                                                │  │
│   │     - 未完成的任务                                                  │  │
│   │                                                                     │  │
│   │     对话历史：                                                      │  │
│   │     ${oldMessages}                                                  │  │
│   │   `                                                                 │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

## 八、取消机制

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           取消机制                                          │
│                                                                             │
│   用户可以随时按 Ctrl+C 取消当前操作                                        │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   AbortController                                                   │  │
│   │                                                                     │  │
│   │   const abortController = new AbortController()                     │  │
│   │   const signal = abortController.signal                             │  │
│   │                                                                     │  │
│   │   // 用户按 Ctrl+C                                                  │  │
│   │   abortController.abort()                                           │  │
│   │                                                                     │  │
│   │   // 各处检查                                                       │  │
│   │   if (signal.aborted) {                                             │  │
│   │     throw new Error('Request was cancelled')                        │  │
│   │   }                                                                 │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   信号传递链：                                                       │  │
│   │                                                                     │  │
│   │   AbortController.signal                                            │  │
│   │         │                                                           │  │
│   │         ├─→ queryLLM(signal)                                        │  │
│   │         │        │                                                  │  │
│   │         │        └─→ API 调用取消                                   │  │
│   │         │                                                           │  │
│   │         ├─→ tool.call(signal)                                       │  │
│   │         │        │                                                  │  │
│   │         │        └─→ 工具执行取消                                   │  │
│   │         │                                                           │  │
│   │         └─→ hooks(signal)                                           │  │
│   │                  │                                                  │  │
│   │                  └─→ Hook 执行取消                                   │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   取消后的处理：                                                     │  │
│   │                                                                     │  │
│   │   yield createAssistantMessage(INTERRUPT_MESSAGE)                   │  │
│   │   // "[Request interrupted by user]"                                │  │
│   │                                                                     │  │
│   │   // 工具执行中的取消                                               │  │
│   │   yield createAssistantMessage(INTERRUPT_MESSAGE_FOR_TOOL_USE)      │  │
│   │   // "[Request interrupted by user for tool use]"                   │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
