# Kode AI Agent 架构总览

> 本文档面向 Go 后端开发者转型 AI Agent 方向，用可视化图表和通俗语言解释 Kode 的整体架构设计。

## 一、Kode 是什么？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   ┌─────────────┐                                                           │
│   │   用户输入   │  "帮我重构这个函数，让它更高效"                            │
│   └──────┬──────┘                                                           │
│          │                                                                   │
│          ▼                                                                   │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │                         Kode AI Agent                               │   │
│   │  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────────────┐    │   │
│   │  │ 理解意图  │→│ 制定计划  │→│ 执行工具  │→│ 验证结果 & 反馈   │    │   │
│   │  └──────────┘  └──────────┘  └──────────┘  └──────────────────┘    │   │
│   │       ↑              ↑              ↑              ↑                │   │
│   │       └──────────────┴──────────────┴──────────────┘                │   │
│   │                        AI 模型持续决策                               │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│          │                                                                   │
│          ▼                                                                   │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  ✓ 读取了 3 个文件    ✓ 重写了 2 个函数    ✓ 运行了测试用例         │   │
│   │  ✓ 提交了代码变更    ✓ 输出了修改摘要                                 │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**Kode 不是一个聊天机器人，而是一个"能动手干活"的 AI 代理。**

核心区别：
| 传统聊天机器人 | Kode AI Agent |
|--------------|---------------|
| 只能回答问题 | 能执行实际操作 |
| 输出是文字 | 输出是代码变更/命令执行 |
| 被动响应 | 主动规划并执行任务 |
| 单轮对话 | 多轮工具调用循环 |

## 四、与 Go 后端架构的对比

作为 Go 开发者，可以这样理解 Kode 的架构：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     Go 后端架构 vs Kode 架构                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Go 后端                           Kode                                     │
│  ────────                          ─────                                    │
│                                                                             │
│  HTTP Handler                     CLI/ACP/MCP 入口                         │
│  (gin.HandlerFunc)                (entrypoints/)                           │
│         ↓                                   ↓                               │
│  Service Layer                    App Layer (app/query.ts)                 │
│  (业务逻辑)                        (查询编排引擎)                            │
│         ↓                                   ↓                               │
│  Repository/DAO                   Tools (tools/)                           │
│  (数据访问)                        (能力执行)                               │
│         ↓                                   ↓                               │
│  Database                         File System + External APIs              │
│  (持久化)                          (外部世界)                               │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  关键区别：                                                                  │
│                                                                             │
│  1. Go: 请求-响应模型               Kode: 循环-迭代模型                     │
│     (一次处理一个请求)                 (持续对话，多轮工具调用)              │
│                                                                             │
│  2. Go: 确定性业务逻辑              Kode: AI 驱动的决策                     │
│     (代码决定做什么)                   (模型决定做什么)                      │
│                                                                             │
│  3. Go: 数据库事务                  Kode: 权限治理                          │
│     (保证数据一致性)                   (保证操作安全性)                      │
│                                                                             │
│  4. Go: 中间件链                    Kode: Hooks 系统                        │
│     (请求预处理/后处理)                (工具执行前后的钩子)                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

│   │                                                                     │  │
│   │   ┌─────────────────────────────────────────────────────────────┐  │  │
