# 权限与安全治理

> 本文档详细解析 Kode 的权限系统，这是 AI Agent 安全运行的核心保障。

## 一、为什么需要权限系统？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         AI Agent 的安全挑战                                 │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   传统软件：                                                        │  │
│   │   ┌─────────────────────────────────────────────────────────────┐  │  │
│   │   │  程序员写的代码 → 确定性操作 → 可预测的结果                  │  │  │
│   │   │                                                             │  │  │
│   │   │  用户点击"删除"按钮 → 代码执行删除 → 文件被删除             │  │  │
│   │   │  一切都在程序员掌控中                                        │  │  │
│   │   └─────────────────────────────────────────────────────────────┘  │  │
│   │                                                                     │  │
│   │   AI Agent：                                                        │  │
│   │   ┌─────────────────────────────────────────────────────────────┐  │  │
│   │   │  用户说"清理一下项目" → AI 决定怎么做 → ???                 │  │  │
│   │   │                                                             │  │  │
│   │   │  AI 可能：                                                   │  │  │
│   │   │  • 删除 node_modules (正确)                                  │  │  │
│   │   │  • 删除 .git (错误但可能)                                    │  │  │
│   │   │  • 删除 / (灾难性)                                           │  │  │
│   │   │                                                             │  │  │
│   │   │  AI 的行为不可完全预测，需要权限系统约束                     │  │  │
│   │   └─────────────────────────────────────────────────────────────┘  │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   权限系统的目标：                                                          │
│   • 防止 AI 执行危险操作                                                   │
│   • 让用户保持对关键操作的最终控制权                                       │
│   • 提供灵活的权限模式适应不同场景                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 二、权限模式

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           权限模式                                          │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   1. default (默认模式)                                             │  │
│   │      ┌─────────────────────────────────────────────────────────┐   │  │
│   │      │  • 每个敏感操作都询问用户                                │   │  │
│   │      │  • 最安全，但需要频繁确认                                │   │  │
│   │      │  • 适合初次使用或处理敏感项目                            │   │  │
│   │      │                                                          │   │  │
│   │      │  [AI 想执行 Bash: rm -rf dist]                           │   │  │
│   │      │  [用户] 允许 / 拒绝 / 总是允许 / 总是拒绝                │   │  │
│   │      └─────────────────────────────────────────────────────────┘   │  │
│   │                                                                     │  │
│   │   2. acceptEdits (接受编辑模式)                                     │  │
│   │      ┌─────────────────────────────────────────────────────────┐   │  │
│   │      │  • 自动允许文件编辑操作                                  │   │  │
│   │      │  • 其他操作仍需确认                                      │   │  │
│   │      │  • 适合大量代码修改场景                                  │   │  │
│   │      │                                                          │   │  │
│   │      │  [AI 想执行 Edit: app.ts] → 自动允许                     │   │  │
│   │      │  [AI 想执行 Bash: npm test] → 询问用户                   │   │  │
│   │      └─────────────────────────────────────────────────────────┘   │  │
│   │                                                                     │  │
│   │   3. bypassPermissions (绕过权限模式)                               │  │
│   │      ┌─────────────────────────────────────────────────────────┐   │  │
│   │      │  • 自动允许所有操作                                      │   │  │
│   │      │  • 危险！仅用于受信任的环境                              │   │  │
│   │      │  • 仍有安全底线检查                                      │   │  │
│   │      │                                                          │   │  │
│   │      │  [AI 想执行任何操作] → 自动允许                          │    │  │
│   │      │  (但会阻止明显危险的操作，如 rm -rf /)                   │   │  │
│   │      └─────────────────────────────────────────────────────────┘   │  │
│   │                                                                     │  │
│   │   4. plan (计划模式)                                                │  │
│   │      ┌─────────────────────────────────────────────────────────┐   │  │
│   │      │  • 只读模式，不允许修改操作                              │   │  │
│   │      │  • 用于分析和规划                                        │   │  │
│   │      │  • 可以创建计划文件                                      │   │  │
│   │      │                                                          │   │  │
│   │      │  [AI 想执行 Read: app.ts] → 允许                         │   │  │
│   │      │  [AI 想执行 Edit: app.ts] → 拒绝                         │   │  │
│   │      └─────────────────────────────────────────────────────────┘   │  │
│   │                                                                     │  │
│   │   5. dontAsk (不询问模式)                                           │  │
│   │      ┌─────────────────────────────────────────────────────────┐   │  │
│   │      │  • 自动拒绝需要询问的操作                                │   │  │
│   │      │  • 用于非交互式场景                                      │   │  │
│   │      │                                                          │   │  │
│   │      │  [AI 想执行需要确认的操作] → 自动拒绝                    │   │  │
│   │      └─────────────────────────────────────────────────────────┘   │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 三、权限检查流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         权限检查流程                                        │
│                                                                             │
│   canUseTool(tool, input, context, assistantMessage)                        │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  1. 检查权限模式                                                    │  │
│   │                                                                     │  │
│   │     if (bypassPermissions && !requiresUserInteraction) {            │  │
│   │       // 检查安全底线                                               │  │
│   │       if (isUnsafePath(path)) return DENY                          │  │
│   │       return ALLOW                                                  │  │
│   │     }                                                               │  │
│   │                                                                     │  │
│   │     if (dontAsk) return AUTO_DENY                                  │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  2. 检查工具特性                                                    │  │
│   │                                                                     │  │
│   │     if (!tool.needsPermissions(input)) return ALLOW                │  │
│   │                                                                     │  │
│   │     某些工具（如只读工具）可能不需要权限检查                        │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  3. 检查权限规则                                                    │  │
│   │                                                                     │  │
│   │     规则优先级：总是拒绝 > 总是询问 > 总是允许                      │  │
│   │                                                                     │  │
│   │     // 检查是否在拒绝列表                                           │  │
│   │     if (matchesRule(input, alwaysDenyRules)) {                      │  │
│   │       return { result: false, shouldPromptUser: false }            │  │
│   │     }                                                               │  │
│   │                                                                     │  │
│   │     // 检查是否在询问列表                                           │  │
│   │     if (matchesRule(input, alwaysAskRules)) {                       │  │
│   │       return { result: false, message: '需要确认' }                │  │
│   │     }                                                               │  │
│   │                                                                     │  │
│   │     // 检查是否在允许列表                                           │  │
│   │     if (matchesRule(input, alwaysAllowRules)) {                     │  │
│   │       return { result: true }                                       │  │
│   │     }                                                               │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  4. 工具特定检查                                                    │  │
│   │                                                                     │  │
│   │     switch (tool) {                                                 │  │
│   │       case BashTool:                                                │  │
│   │         return checkBashPermissions(command, context)              │  │
│   │       case FileReadTool:                                            │  │
│   │         return checkFileReadPermissions(path, context)             │  │
│   │       case FileEditTool:                                            │  │
│   │         return checkFileEditPermissions(path, context)             │  │
│   │       // ...                                                        │  │
│   │     }                                                               │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  5. 返回结果                                                        │  │
│   │                                                                     │  │
│   │     {                                                               │  │
│   │       result: boolean,           // true = 允许, false = 拒绝      │  │
│   │       message?: string,          // 拒绝原因或提示                  │  │
│   │       shouldPromptUser?: boolean, // 是否应该询问用户               │  │
│   │       suggestions?: string[],    // 权限建议                        │  │
│   │     }                                                               │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 四、权限规则

### 4.1 规则格式

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           权限规则格式                                      │
│                                                                             │
│   基本格式：ToolName(args)                                                   │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   工具级别规则：                                                    │  │
│   │   ┌─────────────────────────────────────────────────────────────┐  │  │
│   │   │  Read          - 允许/拒绝所有读取操作                      │  │  │
│   │   │  Bash          - 允许/拒绝所有命令执行                      │  │  │
│   │   │  WebFetch      - 允许/拒绝所有网络请求                      │  │  │
│   │   └─────────────────────────────────────────────────────────────┘  │  │
│   │                                                                     │  │
│   │   参数级别规则：                                                    │  │
│   │   ┌─────────────────────────────────────────────────────────────┐  │  │
│   │   │  Read(/etc/*)        - 匹配 /etc 下的所有文件               │  │  │
│   │   │  Read(/src/**/*.ts)  - 匹配 src 下所有 ts 文件              │  │  │
│   │   │  Bash(npm *)         - 匹配所有 npm 开头的命令              │  │  │
│   │   │  WebFetch(domain:github.com)  - 匹配 github.com 域名        │  │  │
│   │   └─────────────────────────────────────────────────────────────┘  │  │
│   │                                                                     │  │
│   │   通配符规则：                                                      │  │
│   │   ┌─────────────────────────────────────────────────────────────┐  │  │
│   │   │  Read(*)      - 匹配所有路径                                │  │  │
│   │   │  Bash(git *)  - 匹配所有 git 命令                           │  │  │
│   │   │  mcp__github__*  - 匹配 github MCP 服务器的所有工具         │  │  │
│   │   └─────────────────────────────────────────────────────────────┘  │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 五、文件权限检查

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         文件权限检查                                        │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  checkFileReadPermissions(path, context)                            │  │
│   │                                                                     │  │
│   │  1. 路径安全检查                                                    │  │
│   │     • UNC 路径检查 (\\server\share)                                │  │
│   │     • 可疑 Windows 路径检查                                        │  │
│   │     • 符号链接展开                                                  │  │
│   │                                                                     │  │
│   │  2. 规则匹配                                                        │  │
│   │     • 检查拒绝规则                                                  │  │
│   │     • 检查询问规则                                                  │  │
│   │     • 检查允许规则                                                  │  │
│   │                                                                     │  │
│   │  3. 工作目录检查                                                    │  │
│   │     if (path in workingDirectories) return ALLOW                   │  │
│   │                                                                     │  │
│   │  4. 特殊路径检查                                                    │  │
│   │     • 计划文件 (plan mode 允许)                                    │  │
│   │     • 项目指令文件                                                 │  │
│   │                                                                     │  │
│   │  5. 默认：需要确认                                                  │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  checkFileEditPermissions(path, context)                            │  │
│   │                                                                     │  │
│   │  1. 写入安全检查                                                    │  │
│   │     getWriteSafetyCheckForPath(path)                                │  │
│   │                                                                     │  │
│   │     危险路径：                                                      │  │
│   │     • /etc/*                                                        │  │
│   │     • ~/.ssh/*                                                      │  │
│   │     • .env, .env.*                                                  │  │
│   │     • *.pem, *.key                                                  │  │
│   │                                                                     │  │
│   │  2. 规则匹配（同读取）                                              │  │
│   │                                                                     │  │
│   │  3. acceptEdits 模式检查                                            │  │
│   │     if (mode === 'acceptEdits' && path in workingDirs) return ALLOW│  │
│   │                                                                     │  │
│   │  4. 计划模式检查                                                    │  │
│   │     if (planMode && path !== planFile) return DENY                 │  │
│   │                                                                     │  │
│   │  5. 默认：需要确认                                                  │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 六、Bash 命令权限检查

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       Bash 命令权限检查                                     │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  checkBashPermissions(command, context)                             │  │
│   │                                                                     │  │
│   │  1. 安全命令检查                                                    │  │
│   │     isSafeBashCommand(command)                                      │  │
│   │                                                                     │  │
│   │     安全命令（自动允许）：                                          │  │
│   │     • ls, cat, echo, pwd                                            │  │
│   │     • git status, git log, git diff                                 │  │
│   │     • npm list, npm test                                            │  │
│   │                                                                     │  │
│   │  2. 沙箱检查                                                        │  │
│   │     getBunShellSandboxPlan(command, context)                        │  │
│   │                                                                     │  │
│   │     沙箱模式：                                                      │  │
│   │     • shouldBlockUnsandboxedCommand: 必须在沙箱中运行              │  │
│   │     • shouldAutoAllowBashPermissions: 沙箱允许，自动通过           │  │
│   │                                                                     │  │
│   │  3. 危险命令检查                                                    │  │
│   │                                                                     │  │
│   │     危险模式：                                                      │  │
│   │     • rm -rf /*                                                     │  │
│   │     • sudo *                                                        │  │
│   │     • mkfs *                                                        │  │
│   │     • dd if=*                                                       │  │
│   │     • > /dev/sd*                                                    │  │
│   │                                                                     │  │
│   │  4. 规则匹配                                                        │  │
│   │     • 检查 Bash 工具级别的规则                                     │  │
│   │     • 检查命令模式规则                                              │  │
│   │                                                                     │  │
│   │  5. 默认：需要确认                                                  │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  命令分类示例                                                        │  │
```

## 九、与 Go 后端的类比

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     Go 后端 vs Kode 权限系统                                │
│                                                                             │
│   Go 后端                                                                   │
│   ────────                                                                  │
│   • 认证 (Authentication) - 验证用户身份                                   │
│   • 授权 (Authorization) - 验证用户权限                                    │
│   • 中间件 - 在处理请求前检查权限                                          │
│   • RBAC - 基于角色的访问控制                                              │
│                                                                             │
│   Kode                                                                      │
│   ─────                                                                     │
│   • 权限模式 - 类似于角色，决定默认行为                                    │
│   • 权限规则 - 类似于权限策略，细粒度控制                                  │
│   • canUseTool - 类似于中间件，工具执行前检查                              │
│   • ToolPermissionContext - 类似于权限上下文                               │
│                                                                             │
│   核心区别：                                                                │
│   ─────────                                                                 │
│   • Go: 检查"用户"有没有权限做某事                                         │
│   • Kode: 检查"AI"有没有权限做某事                                         │
│                                                                             │
│   • Go: 权限由系统管理员配置                                               │
│   • Kode: 权限由用户实时确认或预先配置                                     │
│                                                                             │
│   • Go: 请求-响应模式，一次检查                                            │
│   • Kode: 循环模式，每个工具调用都要检查                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 十、安全最佳实践

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          安全最佳实践                                       │
│                                                                             │
│   1. 默认拒绝                                                               │
│      • 不在允许列表中的操作默认需要确认                                    │
│      • 宁可多问，不要少问                                                  │
│                                                                             │
│   2. 最小权限                                                               │
│      • 只授予必要的权限                                                    │
│      • 使用具体的规则而非通配符                                            │
│                                                                             │
│   3. 安全底线                                                               │
│      • 即使在 bypassPermissions 模式也要阻止明显危险的操作                 │
│      • 保护系统关键路径                                                    │
│                                                                             │
│   4. 审计日志                                                               │
│      • 记录所有权限决策                                                    │
│      • 便于事后审查                                                        │
│                                                                             │
│   5. 用户教育                                                               │
│      • 清晰解释为什么需要某个权限                                          │
│      • 提供安全的默认选择                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
