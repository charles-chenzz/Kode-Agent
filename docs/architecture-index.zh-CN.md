## 核心概念速查

### AI Agent vs 传统后端

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   传统后端                         AI Agent                                 │
│   ─────────                        ────────                                 │
│                                                                             │
│   请求-响应模型                    循环-迭代模型                            │
│   确定性业务逻辑                   AI 驱动的决策                            │
│   数据库事务                       权限治理                                 │
│   中间件链                         Hooks 系统                              │
│   代码控制流程                     模型控制流程                             │
│                                                                             │
│   核心差异：决策权从程序员转移到了 AI 模型                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 关键术语

| 术语 | 解释 | Go 后端类比 |
|------|------|-------------|
| Agentic Loop | AI 持续决策、执行、观察的循环 | 无（传统是单次请求-响应） |
| Tool | AI 可以调用的能力单元 | Service 方法 |
| ToolUseContext | 工具执行的上下文 | 请求上下文 (context.Context) |
| PermissionMode | 权限控制模式 | RBAC 角色 |
| Hook | 生命周期钩子 | 中间件 |
| MCP | Model Context Protocol，外部工具协议 | gRPC/HTTP API |
| Message | 对话消息（用户/AI/进度） | 请求/响应对象 |
| Session | 会话状态 | 会话/连接 |
